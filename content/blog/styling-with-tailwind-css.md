I started with traditional CSS and frameworks like Bootstrap before trying Tailwind. The utility-first approach felt strange at first — writing long class names in HTML seemed wrong — but after a day of using it, I was hooked.

## What I Learned

- **Utility Classes** — building UIs with small, composable classes instead of custom CSS
- **Responsive Design** — using breakpoint prefixes like `sm:`, `md:`, `lg:` for mobile-first layouts
- **Flexbox & Grid** — layout patterns with Tailwind's flex and grid utilities
- **Customization** — configuring themes, colors, and fonts in the Tailwind config

## Building Real Projects

I used Tailwind CSS for this portfolio. It made building a responsive site fast without writing custom CSS files for every component.

## Where I Am Now

Tailwind is my default for styling. I can build responsive layouts quickly and customize it when needed without fighting the framework.
